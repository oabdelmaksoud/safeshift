# SafeShift Independent Expert-Validation Protocol

The synthetic evaluation (`run_eval.py`, `graph_eval.py`) measures **construct recovery** — how well a
model recovers a target built from the same features. It cannot establish **face validity**: whether
SafeShift's rankings agree with how experienced practitioners judge integration risk. This protocol
runs that independent check. It is execution-ready; the only missing input is real expert ratings.

> **Status:** the pipeline is built and tested end-to-end on `demo_ratings_SYNTHETIC.csv` (random,
> simulated ratings, expert ids `DEMO-*`). **Those demo numbers are not validation and must not be
> cited.** Real validation requires real ratings from the vetted independent experts.

## Roles
- **Study owner** (you): holds the SafeShift score keys, sends blinded sheets, merges returns, runs
  the analysis. Never shares the keys or SafeShift scores with raters until after they submit.
- **Experts** (≥ 3 recommended; Fleiss' κ needs ≥ 3): independent automotive E/E or functional-safety
  practitioners, not involved in building SafeShift.

## Materials (already generated by `build_elicitation.py`)
- `elicitation_<example>_BLINDED.csv` — one per worked example; the interface list with engineering
  attributes (from/to, protocol, signals, safety/timing flags) and **no SafeShift scores**. This is
  what each expert fills in.
- `safeshift_key_<example>.csv` — the SafeShift score/band per interface. **Owner-only; never sent.**
- `demo_ratings_SYNTHETIC.csv` — a fake end-to-end test of the analysis (delete before any real run).

Regenerate any time:
```bash
python evaluation/expert_validation/build_elicitation.py
```

## Procedure
1. **Send** each expert the `*_BLINDED.csv` sheet(s). Instructions to include:
   > For each interface, rate its **integration-defect risk** as HIGH / MEDIUM / LOW in the
   > `your_band` column (optionally a 0–10 score). Judge how likely this interface is to cause
   > integration or architectural trouble when the system is brought together. Rate independently;
   > do not confer with other raters. You are NOT shown any tool's score — use your own judgement.
2. **Collect** the filled sheets (one per expert).
3. **Merge** each returned sheet with the key into the long format the analyser consumes:
   ```python
   from evaluation.expert_validation.build_elicitation import merge_returned
   merge_returned("returned_adas_expertA.csv", "safeshift_key_adas.csv",
                  expert_id="E1", out_csv="expert_ratings.csv")   # repeat per expert/example (appends)
   ```
4. **Analyse**:
   ```bash
   python evaluation/expert_study.py expert_ratings.csv
   ```
   This reports inter-expert concordance (Kendall's W, pairwise Spearman), SafeShift-vs-consensus
   (Spearman ρ, Kendall τ), band agreement (precision/recall/F1, Fleiss' κ), top-5 overlap, and a
   GO / PARTIAL / NO-GO verdict.

## Interpreting the result (pre-registered, do not move the goalposts)
- **GO** — pooled consensus Spearman ≥ 0.6 **and** top-5 intersection ≥ 3/5 in **both** examples
  **and** pooled band F1 ≥ 0.6.
- **PARTIAL** — pooled consensus Spearman in [0.4, 0.6).
- **NO-GO** — otherwise. **A NO-GO is a publishable, honest result** and must be reported as readily
  as a GO; the point of an independent check is that it can fail.

## Honesty notes
- Experts are blinded to SafeShift's scores to avoid anchoring; the thresholds above are fixed in
  advance so the verdict cannot be reverse-engineered.
- This face-validity check is independent of, and does not depend on, the synthetic studies. No
  claim in the paper currently rests on expert data, because none has been collected yet.
